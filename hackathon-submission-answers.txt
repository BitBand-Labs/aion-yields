1. TECHNICAL DOCUMENTATION
==========================

Tech Stack:
- Smart Contracts: Solidity 0.8.20, Hardhat v3, OpenZeppelin contracts
- Chainlink Services: CRE (Compute Runtime Environment), Functions, Automation, Data Feeds, ACE (Automated Compliance Engine)
- Cross-Chain: Avalanche Warp Messaging via Teleporter/ICM (migrated from Chainlink CCIP)
- AI Engine: Python FastAPI server, Anthropic Claude API, Web3.py for on-chain reads
- Frontend: Next.js 15, TypeScript, Framer Motion, Wagmi v2, Reown AppKit (wallet connection)
- External Data: DeFi Llama API for Aave V3 and Compound APY benchmarking

Architecture Decisions:

1. Avalanche C-Chain as primary deployment target: Chosen for EVM compatibility, fast finality (~2s), low gas costs, and native Warp Messaging support. All 20 contracts are deployed and source-verified on SnowTrace.

2. Teleporter/ICM over CCIP for cross-chain: We migrated from Chainlink CCIP to Avalanche's native Warp Messaging (Teleporter) for cross-chain deposits. This gives us validator-secured messaging with zero LINK fees and native Avalanche ecosystem integration. The Lock & Message pattern locks tokens on the source chain and deposits from pre-funded vault reserves on the destination.

3. Kink-based interest rate model with AI optimization: Rather than static rate parameters, our InterestRateModel uses a two-slope curve (like Aave/Compound) where the AI continuously tunes baseRate, slope1, slope2, and optimalUtilization based on real-time market analysis.

4. ACE (Automated Compliance Engine) for AI guardrails: Every AI action passes through on-chain policy validation before execution. CertifiedActionValidatorPolicy requires EIP-712 signed certificates, and VolumeRatePolicy enforces per-action limits (10% TVL max), cumulative limits (500k USDC/window), and frequency limits (5 actions/hour).

5. ERC-8004-inspired Agent Registry: AI agents must register, stake tokens, and maintain reputation scores. This creates accountability — agents can be slashed for bad recommendations.

Implementation Approach:
- Phase 1 (Core): LendingPool with full deposit/borrow/withdraw/repay/liquidate, InterestRateModel, ChainlinkPriceOracle
- Phase 2 (AI + Chainlink): AIYieldEngine, CREExecutionHook (5 workflow types), ChainlinkFunctionsConsumer, LiquidationAutomation
- Phase 3 (Cross-Chain): CrossChainVault with Teleporter on both Sepolia and Fuji, remote vault wiring, token mapping
- Phase 4 (Compliance): PolicyEngine, CertifiedActionValidatorPolicy, VolumeRatePolicy (ACE framework)
- Phase 5 (AI Economy): AIAgentRegistry, X402PaymentGateway, AIRevenueDistributor
- Phase 6 (AI Backend): Multi-chain FastAPI server reading from both Fuji and Sepolia, Claude-powered analysis


2. ARCHITECTURE DESIGN OVERVIEW
================================

Main Components:

A. Money Market Core (LendingPool + InterestRateModel + AToken + VariableDebtToken)
   - Full lending protocol: deposit collateral, borrow against it, earn supply APY
   - Health factor monitoring with liquidation at HF < 1.0
   - Reserve factor for protocol revenue
   - AI-adjustable rate parameters via AIYieldEngine

B. Chainlink CRE Orchestration (CREExecutionHook)
   - Central coordinator for 5 autonomous workflow types:
     1. AI Rate Adjustment — tune interest curve parameters
     2. Liquidation Scan — identify unhealthy positions
     3. Cross-Chain Rebalance — optimize liquidity across chains
     4. Risk Monitoring — assess protocol risk levels
     5. Yield Allocation — distribute capital across protocols
   - Pre-hooks gather on-chain state (reserves, prices, allocator positions)
   - Post-hooks apply AI results to on-chain contracts

C. AI Engine (FastAPI + Claude)
   - Reads live on-chain data from LendingPool and PriceOracle on both chains
   - Fetches external APYs from DeFi Llama (Aave V3, Compound)
   - Claude analyzes utilization, competitive rates, risk factors
   - Produces rate recommendations (baseRate, slope1, slope2, optimalUtilization)
   - Produces allocation recommendations (AION pool %, Aave %, Morpho %)
   - 6 API endpoints: /health, /reserve, /price, /external-apys, /analyze, /predict

D. Cross-Chain Layer (CrossChainVault + Teleporter)
   - Deployed on both Sepolia and Avalanche Fuji
   - Lock & Message pattern: lock tokens on source, Warp message to destination
   - Destination vault auto-deposits into LendingPool for the user
   - Token mapping between chains (source token -> local equivalent)
   - Pre-funded vaults with MockUSDC for immediate destination deposits

E. Compliance Framework (PolicyEngine + ACE Policies)
   - PolicyEngine: central router that loops through registered policies per action
   - CertifiedActionValidatorPolicy: EIP-712 signature verification
   - VolumeRatePolicy: sliding-window rate/volume limits
   - policyCheck modifier on AIYieldEngine and AutonomousAllocator

F. AI Agent Economy (AIAgentRegistry + X402PaymentGateway + AIRevenueDistributor)
   - ERC-8004-inspired agent identity and reputation system
   - Stake-to-participate model with slashing for bad actors
   - x402 payment protocol for AI inference micropayments
   - Revenue distribution: 70% agent pool, 15% top agent bonus, 10% community, 5% protocol

Workflows:

Primary Loop (Autonomous Optimization):
  CRE triggers -> CREExecutionHook gathers on-chain data -> Chainlink Functions routes to AI Engine -> Claude produces recommendations -> CREExecutionHook applies results via AIYieldEngine -> PolicyEngine validates via ACE -> LendingPool rate parameters updated

Cross-Chain Flow:
  User calls depositCrossChain() on source vault -> Tokens locked -> Teleporter sends Warp message -> Destination vault receives message -> Auto-deposits into destination LendingPool -> User earns supply APY on destination chain


3. USER JOURNEY (START TO FINISH)
==================================

Step 1: Connect Wallet
- User visits the AION Yield dashboard (Next.js frontend)
- Clicks "Connect Wallet" which opens Reown AppKit modal
- Selects wallet provider (MetaMask, WalletConnect, etc.)
- Switches network to Avalanche Fuji or Ethereum Sepolia using the network switcher

Step 2: View Markets
- Markets page displays available lending pools with current APYs
- Shows supply APY, borrow APY, total supplied, total borrowed
- APYs are dynamically calculated from the on-chain InterestRateModel
- AI-recommended rates are displayed alongside current rates

Step 3: Supply Assets
- User selects an asset (e.g., MockUSDC) and enters deposit amount
- Approves the LendingPool to spend their tokens (ERC-20 approve)
- Calls deposit() on LendingPool
- Receives aTokens (AToken) representing their deposit — balance grows with interest
- Supply APY starts accruing immediately through the liquidity index mechanism

Step 4: Borrow Against Collateral
- With deposits as collateral, user can borrow other assets
- Dashboard shows available borrow amount based on collateral value and LTV ratio
- Calls borrow() on LendingPool
- Receives borrowed tokens, VariableDebtToken tracks the debt
- Health factor displayed in real-time (liquidation threshold at HF < 1.0)

Step 5: Monitor AI Optimization
- AI Yield Dashboard shows Claude's latest analysis:
  - Current utilization vs optimal target (75-85%)
  - AION rates vs Aave V3 and Compound (competitive positioning)
  - Risk assessment score (1-10)
  - Recommended rate parameter adjustments
  - Cross-protocol allocation recommendations
- Users can trigger manual AI inference from the Agents page
- x402 payment counter shows simulated micropayment cost per inference

Step 6: Cross-Chain Deposits (Optional)
- User on Sepolia can deposit into Fuji LendingPool via CrossChainVault
- Calls depositCrossChain() with destination chain ID
- Tokens locked on Sepolia, Warp message sent via Teleporter
- Destination vault on Fuji auto-deposits into LendingPool
- User earns supply APY on Fuji without manually bridging

Step 7: Manage Positions
- Portfolio page shows all positions across chains
- Withdraw: redeem aTokens for underlying + accrued interest
- Repay: pay back borrowed amount + accrued interest (VariableDebtToken burned)
- Health factor monitoring with warnings as it approaches 1.0

Step 8: Liquidation Protection
- If a user's health factor drops below 1.0, LiquidationAutomation detects it
- Chainlink Automation (checkUpkeep/performUpkeep) monitors positions
- Liquidators can close up to 50% of the undercollateralized position
- Liquidation bonus incentivizes timely liquidation (protocol stays solvent)
